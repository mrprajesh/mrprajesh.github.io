Enable recent gcc version using devtoolset on centOs

This is a trick we use for using various gcc version on the same machine. The machine my advisor runs on centOs. Everyone has a different requirement for gcc. So, they have this wonderful option of devtoolset. Must be installed by root user. Assuming that appropriate version is installed. Lets see how to enable it.

$ g++ --version
g++ (GCC) 4.8.5 20150623 (Red Hat 4.8.5-28)

$ scl enable devtoolset-7 bash

$ g++ --version
g++ (GCC) 7.3.1 20180303 (Red Hat 7.3.1-5)

Usually the installation goes here.
